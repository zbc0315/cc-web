核心设计决策（WHY）：
1. Open-with-continue: 打开/启动已有项目用 --continue 恢复对话，仅新项目不带
2. Per-project .ccweb/: 数据跟项目走，重装不丢失，Open Project 可恢复
3. Session tailing: 读 Claude 原生 JSONL(~/.claude/projects/) 而非解析 PTY 输出，路径编码 /空格_ → -
4. CLI update: SIGUSR2 进 update mode，PTY killed 但 status 保持 running，更新后 resumeAll --continue
5. Activity push: Dashboard 用 /ws/dashboard 替代轮询，TerminalManager/SessionManager 发 EventEmitter
6. Claude Code Hooks: PreToolUse/PostToolUse/Stop → curl POST /api/hooks，HooksManager 幂等 install/uninstall，# ccweb-hook 标记
7. React.lazy named export: ChatView 只有 named export，lazy import 须 .then((m) => ({ default: m.ChatView }))
8. Code splitting: 路由级(ProjectPage/SettingsPage/SkillHubPage) + 组件级(ChatView/OfficePreview/GraphPreview) + 重依赖按需加载
