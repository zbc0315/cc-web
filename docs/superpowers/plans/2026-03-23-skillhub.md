# SkillHub Implementation Plan

> **For agentic workers:** Use superpowers:subagent-driven-development or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a GitHub-based SkillHub for sharing, browsing, and downloading shortcut commands.

**Architecture:** GitHub repo `zbc0315/ccweb-skillhub` stores skills as JSON files with a `skills.json` index. Backend proxies/caches the index and handles Issue creation via bot token. Frontend adds a SkillHub page and share dialog.

**Tech Stack:** Express (backend routes), React + shadcn/ui (frontend), GitHub REST API (Issues + content update)

### Task 2: Backend — SkillHub Routes

**Files:**
- Create: `backend/src/routes/skillhub.ts`
- Modify: `backend/src/index.ts` (mount router at line ~304)

- [ ] **Step 1: Create `backend/src/routes/skillhub.ts`**

Three routes:
- `GET /skills` — fetch `skills.json` from GitHub Raw URL, cache 5 min in memory
- `POST /submit` — create GitHub Issue with bot token (title: `[Skill] {label}`, label: `new-skill`, body: JSON)
- `POST /download/:id` — find skill by id from cached index, add to `global-shortcuts.json` via existing `getGlobalShortcuts()`/`saveGlobalShortcuts()`, update download count in repo via GitHub API

Config: read `CCWEB_GITHUB_TOKEN` from env or `config.json.githubToken`. Repo hardcoded as `zbc0315/ccweb-skillhub`.

- [ ] **Step 2: Mount in `backend/src/index.ts`**

```typescript

// after other app.use lines:

```
