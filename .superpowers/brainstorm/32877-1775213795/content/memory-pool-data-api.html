数据模型 & API 设计

记忆球文件格式、索引结构、后端接口

1. 记忆球文件格式（balls/ball_XXXX.md）

--- id: ball_0042 type: feedback # user / feedback / project / reference B0: 8 # 初始浮力 H: 3 # 累计命中次数 t_last: 156 # 上次被访问的轮次 hardness: 7 # 硬度 (0-10) fusion_potential: 0.3 # 融合潜力 (0-1) links: - target: ball_0015 strength: strong - target: ball_0038 strength: weak created_at: "2026-04-03T10:00:00Z" --- 不在代码中添加多余注释。用户明确表示 不喜欢冗余的 JSDoc 和行内注释，除非 逻辑确实不直观。

2. state.json

{ "t": 156, // 当前全局轮次 "lambda": 0.97, // 衰减率 "alpha": 1.0, // 查询增益 "active_capacity": 20, // 活跃层容量 "next_id": 43, // 下一个球 ID "pool": "project", // 池类型 "initialized_at": "2026-04-03T10:00:00Z" }

3. index.json（前端读取用，AI 每次操作后更新）

{ "t": 156, "updated_at": "2026-04-03T12:30:00Z", "balls": [ { "id": "ball_0042", "type": "feedback", "summary": "不在代码中添加多余注释", "B0": 8, "H": 3, "t_last": 156, "buoyancy": 8.2, // 当前计算浮力 "hardness": 7, "links": ["ball_0015", "ball_0038"] }, ... ] }

4. 后端 API

端点 说明
POST /api/memory-pool/:projectId/init 初始化：创建目录、生成 SPEC/QUICK-REF/state/index、追加 CLAUDE.md
GET /api/memory-pool/:projectId/status 返回 { initialized: bool, state?, ballCount? }
GET /api/memory-pool/:projectId/index 返回 index.json（前端列表/浮球渲染用）
GET /api/memory-pool/:projectId/ball/:ballId 返回单个球的完整内容（YAML frontmatter + 正文）
POST /api/memory-pool/:projectId/command 快捷操作：{ action: "maintain" | "load" | "save" | "general" } → writeRaw 到终端

注意：所有写操作（创建/修改/删除球）由 Claude CLI 在终端中完成，后端不提供写 API。后端只负责读取文件和发送终端指令。

5. CLAUDE.md 追加内容（初始化时写入）

## 记忆池（Memory Pool） 本项目已启用记忆池系统。 **每次对话开始时：** 1. 读取 `.memory-pool/QUICK-REF.md` 了解操作规范 2. 读取 `.memory-pool/state.json` 获取当前轮次 3. 读取 `.memory-pool/index.json` 加载活跃层记忆 4. 将活跃层记忆纳入当前对话上下文 **对话过程中：** - 遇到重要信息时主动提议存入记忆池 - 用户要求记忆操作时参照 QUICK-REF.md 执行 - 每次操作后更新 index.json **完整规范：** `.memory-pool/SPEC.md`
