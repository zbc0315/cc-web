记忆池设计方案总览

请确认以下设计是否准确

1. 架构概览

ccweb 前端（UI 层）
左面板 Tab：记忆池入口
列表视图（浮力排序）
浮球弹窗（全景概览）
快捷按钮 → 终端指令
→
ccweb 后端（API 层）
GET 读取 .memory-pool/ 文件
POST 初始化记忆池
终端指令注入（writeRaw）
→
Claude CLI（执行层）
读取 SPEC.md 理解规范
创建/修改记忆球文件
执行衰减/分化/融合
更新 state.json 轮次

2. 文件结构

your-project/
├── .memory-pool/
│ ├── SPEC.md ← 完整规范（浮力公式、文件格式、操作流程）
│ ├── QUICK-REF.md ← 精简快速参考（AI 日常操作用）
│ ├── state.json ← 全局状态（轮次计数器 t、参数配置）
│ ├── index.json ← 所有球的索引（浮力排序，供前端读取）
│ └── balls/
│ ├── ball_0001.md ← 记忆球文件（YAML frontmatter + 正文）
│ ├── ball_0002.md
│ └── ...
├── CLAUDE.md ← 追加记忆池入口指令
└── ...

3. 初始化流程

1 用户点击左面板"记忆池"Tab → 显示"初始化记忆池"按钮
2 POST /api/memory-pool/:projectId/init → 创建 .memory-pool/ 目录结构
3 生成 SPEC.md + QUICK-REF.md + state.json + index.json
4 向 CLAUDE.md 追加记忆池指令（如已存在则跳过）
5 前端切换到列表视图，开始轮询 index.json

4. 快捷操作按钮

🧹 整理记忆
📖 读取记忆
💾 保存记忆
🧠 记忆池
整理记忆 → 终端发送：执行记忆池维护（衰减计算、分化、融合）
读取记忆 → 终端发送：读取活跃层记忆到上下文
保存记忆 → 终端发送：从当前对话提取信息存入记忆池
记忆池 → 终端发送：通用指令，AI 自行判断操作

5. 核心参数

轮次计数 t 每条用户消息 +1（消息级）
衰减率 λ 项目池默认 0.97
查询增益 α 默认 1.0
活跃层容量 默认 20 个球（浮力 Top-N）
作用域 项目级（预留全局池扩展）
